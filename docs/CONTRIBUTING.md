# Contributing Guide

## How to Contribute
- Fork the repository
- Create a feature branch
- Write clear, modular code with comments
- Add/modify tests in `tests/`
- Ensure all checks pass (lint, test, security)
- Submit a pull request with a clear description

## Code Style
- Follow PEP8 and Clean Code principles
- Use meaningful names and docstrings
- No hardcoded secrets or credentials

## Ownership
- Each module should have a clear owner (add to module docstring)

## Reporting Issues
- Use GitHub Issues for bugs and feature requests

## Security
- Report vulnerabilities as described in [SECURITY.md](../SECURITY.md)
